1Realms Objects#

1.1Abstract Operations#

1.1.1CreateSpawnRealm(parentRealm)#

The abstract operation CreateSpawnRealm with argument parentRealm performs the following steps:
  1. Assert: parentReam is an instance of a concrete subclass of Realm.
  2. Let childRealm be ? OrdinaryCreateFromConstructor(NewTarget, "%RealmPrototype%", « [[Realm]] »).
  3. Let freshGlobal be ? ObjectCreate(parentRealm.[[Realm]].[[GlobalEnv]]).
  4. Let childRealmRec be CreateRealm().
  5. Let newContext be a new execution context.
  6. Set the Function of newContext to null.
  7. Set the Realm of newContext to childRealmRec.
  8. Set the ScriptOrModule of newContext to null.
  9. Perform SetRealmGlobalObject(childRealmRec, freshGlobal, undefined).
  10. Set childRealm's [[Realm]] internal slot to childRealmRec.
  11. Return childRealm.
Note TODO: Figure a way to make this a lightweight realm.

1.1.2Realm Evaluator Eval Functions#

When a realm evaluator eval function is called with optional argument src, the following steps are taken:
  1. Let theRealm be the this value.
  2. Assert: theRealm is an instance of a concrete subclass of Realm.
  3. If src is not provided, return undefined.
  4. Return ? PerformEval(src, theRealm.[[Realm]], false, false).
The length property of a Number format function is 1.

1.1.3CreateSpawnEvaluators(theRealm)#

The abstract operation CreateEvaluators with argument theRealm performs the following steps:
  1. Assert: theRealm is an instance of a concrete subclass of Realm.
  2. Let freshGlobal be theRealm.[[Realm]].[[GlobalEnv]].
  3. Let E be a new built-in function object as defined in Realm Evaluator Eval Functions (1.1.2).
  4. Let evalFn be BoundFunctionCreate(F, theRealm, « »).
  5. Perform ! DefinePropertyOrThrow(evalFn, "length", PropertyDescriptor {[[Value]]: 1, [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true}).
  6. Perform ? CreateDataPropertyOrThrow(freshGlobal, "eval", evalFn).
  7. TODO: Function evaluator
Note The eval property of the freshGlobal has no relationship with the parent realm's eval, or the realm's eval. Is this a problem?

The value of the [[Prototype]] internal slot of the Realm constructor is the intrinsic object %FunctionPrototype%.

Specifically, it contains no host-specific objects. This frozen global object is a plain object whose [[Prototype]] is Object.prototype, i.e., the %ObjectPrototype% intrinsic of that immutable root realm. Note 2 Since two immutable root realms are forever the same in all ways except object identity, we leave it implementation-defined whether Realm.immutableRoot() always creates a fresh one, or always returns the same one. On any given implementation, it must either be always fresh or always the same.

1.3Properties of the Realm Prototype Object#

1.3.1Realm.prototype.spawn([ endowments ])#

When the Realm.prototype.spawn function is called with optional argument _endowments, the following steps are taken:
  1. Let parentRealm be this value.
  2. If Type(parentRealm) is not Object, throw a TypeError exception.
  3. If parentRealm does not have an [[Realm]] internal slot, throw a TypeError exception.
  4. Let childRealm be ? CreateSpawnRealm(parentRealm).
  5. Let freshGlobal be childRealm.[[Realm]].[[GlobalEnv]].
  6. TODO: spawn populates this freshGlobal with overriding bindings for the evaluators that have global names (currently only eval and Function). It binds each of these names to fresh objects whose [[Prototype]]s are the corresponding objects from the parent realm.
  7. Perform ? CreateSpawnEvaluators(childRealm).
  8. If endowments is provided, then
    1. If Type(endowments) is not an Object, throw a TypeError exception.
    2. Let props be ? ToObject(endowments).
    3. Let keys be ? props.[[OwnPropertyKeys]]().
    4. Repeat for each element propName of keys in List order,
      1. Let propDesc be ? props.[[GetOwnProperty]](propName).
      2. If propDesc is not undefined and propDesc.[[Enumerable]] is true, then
        1. Let propValue be ? Get(props, propName).
        2. Perform ? CreateDataPropertyOrThrow(freshGlobal, propName, propValue).
  9. Return childRealm.
